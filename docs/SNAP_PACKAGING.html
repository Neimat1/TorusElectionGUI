Snap Packaging

This repository includes a basic Snapcraft setup for building an Ubuntu snap package.

What Snap Changes

Snaps are different from .deb packages:

  • A snap is mounted read-only under /snap/<snap-name>/<revision>/.
  • Snap commands are exposed under /snap/bin/.
  • Snaps update automatically through snapd when installed from the Snap Store.
  • User data is isolated under the user's snap data directory.
  • Removing a snap removes the app, while snapshots/user data follow snapd behavior.

Files

  • snapcraft.yaml: Snapcraft build and metadata file.
  • snap/gui/torus-election-gui.desktop: desktop launcher entry.
  • snap/local/torus-election-gui: wrapper script that starts the bundled Java runtime.
  • assets/icons/torus-election-gui.png: snap and desktop icon.

Build Locally

Install Snapcraft:


sudo snap install snapcraft --classic

Build the snap from the repository root:


snapcraft

The output will be a .snap file in the repository root.

Install Locally

For a locally built strict snap:


sudo snap install ./torus-election-gui_1.0.3_amd64.snap --dangerous

Run it:


torus-election-gui

or:


snap run torus-election-gui

Remove it:


sudo snap remove torus-election-gui

Remove it and ask snapd to purge its data:


sudo snap remove --purge torus-election-gui

Logs In Snap

The snap sets:


XDG_DATA_HOME=$SNAP_USER_DATA/.local/share

So the app log is written inside the snap user data area:


~/snap/torus-election-gui/current/.local/share/torus-election-gui/logs/torus-election-gui.log

License

The snap is published under the same open-source license as the repository:


license: MIT

The root LICENSE file contains the MIT License text so GitHub and Snapcraft can both identify the license correctly.

Links

The snap metadata supports these link fields:

  • website
  • source-code
  • issues
  • contact

The current snapcraft.yaml uses placeholder links for website, source-code, and issues. Replace them before publishing to the Snap Store.

Snap Store "last updated" dates are generated by the store after each upload/release. They are not controlled directly by snapcraft.yaml.

Publish To Snap Store

Create or log into a Snapcraft account:


snapcraft login

Register the snap name once:


snapcraft register torus-election-gui

Upload the built snap:


snapcraft upload torus-election-gui_1.0.3_amd64.snap --release=edge

Promote to stable after testing:


snapcraft release torus-election-gui <revision> stable

After publishing, users can install it with:


sudo snap install torus-election-gui
